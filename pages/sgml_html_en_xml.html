SGML, HTML en XML

Ontstaan van SGML

SGML zoals we het nu kennen, is het resultaat van een groep mensen en een samensmelting van verschillende activiteiten, die de afgelopen decennia hebben plaatsgevonden. In de jaren '60 begon het met het concept van generic coding, waarbij je ipv control codes of macros, descriptive tags ging gebruiken voor documenten. Er ontstond eind 1960 een project dat leidde tot een GenCode Comittee, wat later een belangrijke rol zou hebben voor het ontstaan van SGML.

In 1969 leidde Charles Goldfarb een IBM onderzoek naar informatie systemen bij advocatenkantoren. Samen met zijn team ontstond toen de Generalized Markup Language (GML) als middel om de informatie systemen documenten te laten delen.

GML was gebaseerd op het Generic Coding concept van Rice en Tunnicliffe. Maar in plaats van een simpel tag-schema, introduceerde GML het concept van formeel-gedefinieerde document types met een element structuur. Nadat GML af was, ging Goldfarb door met onderzoek doen en het toevoegen van extra concepten, zoals als references, link processen, en concurrent document types. Deze concepten werden later onderdeel van SGML.

In 1978 vroeg de American National Standards Institute (ANSI) committee of Goldfarb erbij wilde om te werken aan een nieuw project, namelijk het creeëren van een text description language standard, gebaseerd op GML. In 1980 werd hieruit toen de eerste werkende versie van SGML gepubliceerd. Zo werd SGML rond de tijd van 1984 een erkend standaard door de ISO en de ANSI. In 1985 werd er vervolgens een eerste international standard uitgegeven van SGML. (ISO 8879:1986)

Hierna werd SGML vooral gebruikt in applicaties voor grote bedrijven. Hierdoor bleef SGML zich ook steeds ontwikkelen tot hoe het nu bekend is. Toen, in 1992, besloten onderzoekers om een Hypertext Markup Language te bouwen, genaamd HTML.

Van SGML naar HTML

In 1989 kreeg HTML leven toen Tim Berners-Lee het Web had uitgevonden met HTML als publish language. De HTML versie die Tim 'uitgevonden' had, was sterk gebaseerd op SGML. SGML kan namelijk geimplementeerd worden op elke machine. Het mooie is dus dat de taal onafhankelijk is van de formatter (browser of andere software die tekst weergeeft). Hierbij werd het gebruik van tags overgeërfd.

Bij HTML kwam hierbij dan wel iets anders, wat nog niet eerder kon bij SGML, namelijk: hypertext links. Het idee van het gebruik van een anchor element met een HREF attribuut was de uitvinding van Berners-Lee.

Zo is HTML een lange tijd door gegaan met ontwikkelen als een applicatie en subset van SGML. SGML heeft immers bewezen dat het zinvol is, zeker aangezien het erkend is als een internationaal standaard. SGML had een flinke reputatie opgebouwd en had erg veel gebruikers, zo had SGML een erge grote invloed dus. HTML kan alleen maar profiteren door mee te gaan met de stroom van SGML's filosofie.

De DTD speelt hier ook een erg grote rol in, deze geeft niet alleen antwoord op veel vragen die syntax-gerelateerd zijn, maar het zorgt er ook voor dat een HTML document gecheckt kan worden. Een HTML versie zonder een DTD is als een taal zonder een woordenboek: Niet iedereen die de taal spreekt heeft een woordenboek nodig, maar diegene die invloed hebben op de ontwikkeling van de taal, kunnen weinig zonder een boek met de juiste referenties.

SGML was gemaakt om document portabiliteit en makkelijke transformatie mogelijkheden te garanderen. De missie van SGML is om te garanderen dat de content die we maken, voor iedereen toegankelijk is, zonder je zorgen te maken of de technologie die je wilt gebruiken wel compatible is.

Om deze redenen bleef HTML dus gebaseerd op SGML, maar dit veranderde toen de nieuwste versie van HTML verscheen: HTML5.

SGML en HTML5

Alhoewel de syntax van HTML veel weg heeft van een SGML-subset, heeft HTML5 ervoor gezorgd dat het zichzelf geen applicatie meer kan noemen van SGML, door hun eigen parser regels te definiëren.

Wel hebben ze ervoor gezorgd dat het backwards compatible is met oudere versies van HTML. De tag met DOCTYPE origineert van SGML en werd in vorige versies van HTML gebruikt als een referentie naar een Document Type Definition - een formele declaratie van de elementen, attributen en syntactische kenmerken die gebruikt kunnen worden in het document.

Nu HTML5 formeel gezien niet meer gebaseerd is op SGML, heeft de DOCTYPE niet langer deze functie, en refereert dus niet meer naar een DTD. De DOCTYPE triggert no-quirks mode in de browsers, waar HTML5 er 3 van kent: Quirks mode, limited quirks mode en no quirks mode.

Verder gaan we niet dieper in op HTML5, aangezien dat niet ons doel is, op andere websites is veel te vinden over hoe HTML5 precies in elkaar zit.

Van SGML tot XML

Het ontstaan van de Extensible Markup Language (XML) is weer een uitwerking van het werk van Charles Goldfarb toen hij de Standardised Generalised Markup Language (SGML) creeërde. Zoals bekend is SGML zelf geen mark-up taal, maar wordt het gebruikt om woordenboeken te maken die documenten kan noteren met structurele tags.

Toen de HyperText Markup Language (HTML) ontworpen werd door Tim Berners Lee aan het einde de tachtige jaren, werd dit beschouwd als de meest populaire applicatie van SGML. Sinds het is ontworpen, is HTML eigenlijk slachtoffer geworden van zijn eigen populariteit, aangezien ze zo snel ontwikkelde en uitbreidde, dat ze hun originele visie meer konden bereiken. Tot op de dag van vandaag blijft HTML erg populair, als een presentatie technologie, maar het wordt wel verklaard als ongeschikt als een data storage format. Als het aankomt op het data storage, dan is HTML hiervoor ongeschikt, want het is origineel gezien bedoelt voor presentatie mogelijkheden. SGML is dan weer te complex in gebruik voor dit doel. Hierdoor is dan XML gemaakt.

De Extensible Markup Language (XML) wordt gezien als de brug tussen HTML en SGML, aangezien XML leesbaar is voor mens en machine, terwijl het flexibel genoeg blijft om platform en architectuur onafhankelijk, data uitwisseling ondersteunt. Er zijn twee manieren om de structuur van een XML document te beschrijven. Dit is te doen met Data Type Definition (DTD) documenten of met een XML schema. De DTD is geïntroduceerd door SGML, terwijl een XML schema is geschreven met gebruik van de XML syntax. Beide mogelijkheden geven specificaties over constraint regels die toegepast zijn op een XML document.

De Document Type Definition is meer gepast bij het gebruik van HTML, DTD kan namelijk de structuur van een document definiëren, maar het heeft geen regels die toegepast kunnen worden als het om data gaat. DTD ziet al de data in een document als een string. Dit is natuurlijk perfect voor een document markup language, maar niet als je de data in het document wil beheren.

Om dit gebrek aan te vullen is het XML Schema Language gecreeërd in 2000. Een XML schema definieert veel meer data types en het geeft specificaties die niet alleen toegepast worden op de structuur van het XML document, maar ook op de inbegrepen data. Hiermee kun je bijvoorbeeld een XML tag definiëren met het type: nonNegativeInteger. Een geldige XML parser zorgt dan voor een foutmelding van het document als de data geen integer is die groter is dan 0.
